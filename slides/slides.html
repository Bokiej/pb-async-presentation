then(..., onReject) method

When error is thrown before then(..., onReject) method it will catch it and won't invoke then() body and pass undefined to next then(). catch() will intercept second error. If catch() will be before onReject then onReject won't intercept error. finally() method will always be invoked both after success or error in Promise chain. Below is how will log from browser console look like. 

                    
                        1. setBulbColor
                        Error: Error 1
                        Error: Error 2
                        finally method

When use onReject inside then() method?

In most cases is better to use single catch() at the end of Promise. But such technique can be used as sophisicated 'if' statement. 'if error occours do something and return something else'.
