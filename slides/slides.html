@wire

In bulbsComponent are imported two asynchronous callbacks. One is js mokup lightBulb Apex like callback which were been used before and getBulb for @wire which returns light bulb simple object.

When bulbs-app runs it is easy to see that @wire and promises inside connectedCallback are execute separately.

@wire is independent from other callbacks

@wire is external asynhronous callback and while we are quering in connectedCallback or somewhere else we can't rely on @wire response because we don't know when it will be resolved.

When to use @wire

  • when we get simple cached data
  • when @wire attributes relays on single constantly changing value and must be refreshed. Then use '$valueName' string pattern.
  • get Salesforce object common data like getObjectInfo

But seems like the last reason is most important.
