How single bulb works in project?

bulbsContainer component

bulbsContainer is component where all bulbs will be rendered. For now there is only one bulb inside to introduce all methods used to show bulbs features. Methos are imported from bulbUtilities.js file inside js inner directory of bulbsContainer component.

Names of bulb methods

Methods in below slide will be invoked for each bulb separately. setBulbColor must be first method in chain, other methods order can be mixed. They will be invoked after async method returns data.

  • setBulbColor({ context, name, color }) will change bulb color and return reference to bulb component
  • setBulbCounter(bulb) will add bulb to array to get it's index in the future to set order number and return reference to bulb component
  • setBulbNumber(bulb) will render bulb order number and return reference to bulb component
  • setBulbTime(bulb) will render bulb end time and return reference to bulb component
  • showAllBulbNumbers({ context }) will search all references to bulb components and show all bulb numbers in one time

lightBulb promise

lightBulb Promise is async method similar to all Apex methods used in Salesforce. It get object with bulb parametters as attribute and returns the same object. Every then block will invoke different bulb method described before. catch block will show error if occours. finally block will render in console string message 'everything is done' both after successful or error Promise chain end.
