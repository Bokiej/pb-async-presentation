Promise one after another

Every error thrown in any promise in a chain will be intersept by catch() method. Console log in browser will look like below. 

                    
                        'yellow' bulb works
                        'red' bulb works
                        Error: Red bulb Error

When to use Promises one after another?

Every time there is necessity to achieve the desired sequence in queries. But this is rather seldom situation.

If we query two Apex callbacks one after another is better to create one callback in Apex rather to run two queries one after another in front-end. It will take longer response if you query many callbacks in frontend.
